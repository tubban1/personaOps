# personaOps Data Structure Registry (V4.2)

## 1. 核心域 (Core Domain)
*   **Persona**: 角色本体模型。
*   **Brand**: 品牌资产与约束模型。
*   **BrandBinding**: 建立人设与品牌的关联权值。

## 2. 策略域 (Strategy Domain)
*   **ContentPillar**: 内容支柱分配（Lifestyle, Pro, Conversion）。
*   **Storyline**: 剧情线容器。
*   **StoryBeat**: 具体剧情节奏点。
*   **WeeklyPlan / PostPlan**: 待执行的内容计划。

## 3. 互动与记忆域 (Interaction & Memory)
*   **InteractionRecord**: 標準化捕获粉丝互动内容、意图与风险等级。
*   **LeadMark**: 业务线索标记（针对高意向咨询）。
*   **HandoffDecision**: 人工接管决策流水。
*   **MemorySnapshot**: (Ongoing) 角色经历的持久化记录。

## 4. 执行域 (Execution Domain - Migrated)
*   **CaptionSchema**: 结构化内容发布契约（Title/Body/Hashtags）。
*   **PlatformPolicy**: 平台发布与裁剪规则。
*   **AuditRecord**: 执行全链路审计（渲染成功与否、发布回执）。
